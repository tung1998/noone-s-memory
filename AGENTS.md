# Repository Guidelines

## Structure
- `Raw/`: draft chapters and manuscript files.
- `Edited/`: revised chapters that are considered more stable.
- `Information/`: canon, timeline, worldbuilding, character notes, and reference material.
- `SUMMARY.md`: table of contents and reading order.
- `.agents/`: detailed Codex workflows, checklists, templates, and policies.

## Core rules
- New story content goes into `Raw/` unless the user explicitly asks to edit `Edited/`.
- Treat `Information/` as canon source of truth unless the user explicitly wants a retcon.
- If story changes introduce or alter canon, update the matching file in `Information/` in the same change.
- If you add or rename reference files, update `SUMMARY.md`.
- Keep terminology consistent across files, especially `Memnet`, `Memlink`, `MIC Zone`, `Mnemonic Shard`, and `Seiromemdas`.
- Preserve the established tone: controlled, eerie, memory-centered, and psychologically tense.
- Do not silently resolve major lore ambiguity or mystery threads too early.

## Before writing
- Read the relevant files in `Information/` first.
- If the task touches timeline, world logic, factions, technology, or character history, verify against `Information/Timeline.md`, `Information/World.md`, and the related note files.

## Working style
- Keep one H1 per Markdown file.
- Use readable Markdown with short sections and clear headings.
- For lore files, prefer concise, scannable structure.
- For chapter prose, prioritize narrative flow over wiki-style formatting.
