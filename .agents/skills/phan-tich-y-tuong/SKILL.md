# Phan Tich Y Tuong

Assess an idea before it hardens into canon. Judge it against the current project, not against generic storytelling advice. Prefer clear verdicts and concrete revision paths over polite vagueness.

## Workflow

1. Read only the minimum canon needed to evaluate the idea.
2. Restate the proposal in one tight block so the judgment is anchored to the same reading.
3. Separate what is genuinely strong from what only sounds interesting on paper.
4. Check for canon conflict, tone drift, mystery damage, and downstream plot cost.
5. Recommend `giu nguyen`, `sua roi dung`, or `khong nen dung`.
6. If revision is needed, improve the core dramatic move instead of only polishing wording.

## Read First

Read only what the idea actually touches:

- `Information/DevelopmentProfile.md` when the idea affects premise, arcs, endgame, or the role of key factions.
- `Information/Timeline.md` when dates, age, sequence, or historical causality matter.
- `Information/World.md` when the idea changes rules, technology, institutions, or social effects.
- The directly relevant note in `Information/Character/` or `Information/Mindmap/` when the idea touches a character, object, or concept already in play.
- The nearby file in `Raw/`, `Edited/`, or `Structure/` when the idea plugs into an existing chapter line.

## Inputs To Expect

Work from one or more of these:

- a lore proposal;
- a reveal or twist;
- a character backstory addition;
- a worldbuilding rule;
- a chapter premise;
- a retcon candidate;
- a user asking whether an idea is good, fits the story, is too early, too much, or not sharp enough.

If the input is vague, infer a usable version of the idea before judging it. State that inferred reading in one short block.

## Evaluation Frame

Judge the idea on these axes:

- `luc kich tinh`: Does it create pressure, consequence, or a meaningful shift, or is it only decorative?
- `do hop canon`: Does it fit existing facts, or does it force silent retcons?
- `do hop tone`: Does it keep the cold, controlled, memory-centered unease of the project?
- `do hop mystery`: Does it deepen the wrongness, or does it explain too much too soon?
- `do ben duong dai`: Does it open useful future consequences, or does it create cleanup problems?

Treat hard contradiction and soft risk differently:

- Hard contradiction: directly breaks timeline, rules, locked roles, or prior scenes.
- Soft risk: technically possible, but weakens tone, flattens mystery, or makes later arcs less interesting.

## Output Pattern

Unless the user asks for a different format, respond in this order:

- `Ket luan`: `manh`, `dung duoc neu sua`, or `khong nen dung`.
- `Tom tat y tuong`: one short restatement.
- `Diem an tien`: what is genuinely working and why.
- `Van de hoac rui ro`: continuity conflict, tone drift, pacing damage, or logic weakness.
- `Ban tot hon`: one stronger rewrite that preserves the user's core intention when possible.
- `Muc anh huong canon`: `thap`, `trung binh`, or `cao`.
- `Can cap nhat gi neu chot`: note any files in `Information/` or `Structure/` that would need updating.

When there is a direct canon conflict, cite the relevant file names instead of speaking vaguely.

## Judgment Style

- Be direct. Do not praise a weak idea just because it is new.
- Be fair. Do not reject an unusual idea only because it is risky if it can be reframed into a stronger version.
- Preserve the user's core desire when suggesting revisions.
- Prefer structural fixes over cosmetic rewrites.
- Distinguish `y tuong chua du tot` from `y tuong sai canon`.
- Mark clearly when something is only a preference call, not a hard continuity issue.

## Quality Bar

Prefer ideas that:

- reveal character through pressure instead of explanation;
- create a small but undeniable wrongness;
- increase consequence without collapsing mystery too early;
- make `Memnet`, memory, identity, or control feel more morally difficult;
- leave room for later escalation.

Push back on ideas that:

- explain the central secret too cleanly;
- add lore that behaves like a cheat code;
- force characters to act against their established logic without cost;
- turn the project into generic dystopia instead of memory-specific tension;
- solve one short-term beat by damaging the long game.

## Handoff

If the idea becomes canon, note the exact follow-up surface:

- character note;
- concept note;
- timeline entry;
- structure file;
- chapter draft;
- `SUMMARY.md` only if a new reference file is created.
