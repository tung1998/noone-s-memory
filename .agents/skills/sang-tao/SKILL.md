# Sang Tao

Generate strong story options without prematurely hardening canon. Work from the repo's existing tone: cold, controlled, psychologically tense, memory-haunted, and slightly wrong in concrete ways.

## Workflow

1. Read the minimum canon needed before proposing anything.
2. Identify what the user already fixed and what is still open.
3. Generate options with different dramatic functions instead of cosmetic rewrites.
4. Mark continuity risks before presenting anything that could affect canon.
5. Keep the chosen option easy to hand off into drafting or expansion.

## Read first

Read only what is necessary:

- `Information/Timeline.md` when time order matters.
- `Information/World.md` when technology, organizations, or rules matter.
- The directly relevant character or concept note when the idea touches established canon.
- The nearby chapter draft in `Raw/` or `Edited/` when the new idea plugs into an existing scene.

## Inputs to expect

Work from one or more of these:

- a premise;
- a chapter role;
- a scene node;
- a character desire or fear;
- a conflict;
- a required reveal;
- a mood or image the user wants to keep.

If the input is vague, infer a usable working brief instead of stopping. State the inferred brief in one short block before proposing options.

## Output pattern

Default to 3 options unless the user asks otherwise. For each option, provide:

- a short label;
- the core dramatic move;
- what changes for the character by the end;
- what mystery, tension, or unease it increases;
- one continuity risk or constraint if relevant.

After the options, recommend one option when there is a clearly strongest fit. Keep the recommendation concrete and tied to tone, pacing, and canon safety.

## Idea quality bar

Prefer options that:

- reveal character through pressure instead of explanation;
- embed lore through behavior, objects, interfaces, or sensory detail;
- create a small but undeniable wrongness;
- leave room for later mystery instead of answering too much;
- move the chapter function forward.

Avoid options that:

- dump exposition;
- resolve the central mystery too early;
- make characters behave outside established motives without cause;
- introduce major lore as if it were already settled canon.

## Common requests

When the user asks for chapter ideation, propose chapter functions and scene beats.

When the user asks for scene ideation, focus on conflict, sensory anchor, and the point of distortion.

When the user asks for "more creative" or "darker" variants, push the psychological pressure and specificity, not only the volume of events.

## Handoff

If the user picks one option, convert it into a compact node list that `trien-khai` can expand next. Include:

- opening image;
- immediate objective;
- escalation;
- point of distortion;
- turn or reveal;
- exit image.
